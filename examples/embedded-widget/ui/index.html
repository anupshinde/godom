Embedded Widget Demo

This page demonstrates godom components embedded in an external website. The page is served by a plain Go static file server on port 9090. The godom components (stock ticker and marquee) run on a separate godom server on port 9091.

godom embedded widget example

How It Works

The host page includes two script tags: one sets GODOM_WS_URL pointing to the godom server, the other loads godom.js from it. godom finds g-component elements and wires them up over WebSocket.

Architecture

Two Servers, One Page

Port 9090 serves this static HTML page. Port 9091 runs the godom engine with live components. The browser connects to both — static content from one, live updates from the other.

Setup

No Mount Needed

The godom engine uses Register() only — no root component. Components attach to g-component elements wherever they appear on the host page.

API

Live Data Over WebSocket

Stock prices update every 500ms via the godom WebSocket connection. The host page doesn't need any JavaScript — godom handles the DOM updates automatically.
