What this demo shows

  • Fully self-contained tool package — Go code, HTML (via AssetsFS), AND the plugin JS all live in tools/solar/. The package exports solar.Plugin so main only has to eng.Use(solar.Plugin) and eng.Register(solar.New()) — no paths, no FS layout knowledge from main.
  • Custom plugin — canvas-bridge.js is embedded with //go:embed canvas-bridge.js and registered via RegisterPlugin("canvas3d", js). The template uses g-plugin:canvas3d="SolarSystem" to hand Go-computed draw commands to the canvas bridge every frame.
  • All 3D math in Go — camera projection, body positions, diffuse lighting. The bridge just paints the commands.
  • <info-note> — same shared partial, same <g-slot/> pattern as the other tools.
